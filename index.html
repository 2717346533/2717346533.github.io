NoSQL：数据与数据之间没有关联关系

SQL ：表与表之间建立关联关系

Nosql四大分类：

  1. 键值存储
  2. 列存储
  3. 文档型（MongoDB）
  4. 图形数据库

NoSQL数据库在以下情况适用：

  1. 数据模型比较简单
  2. 需要灵活性更强的IT系统
  3. 对数据库性能要求较高
  4. 不需要高度的数据一致性
  5. 对于给定key，比较容易映射复杂值的环境

优点：

  1. 丰富的数据结构
  2. 高速读写

缺点：

  1. 持久化
  2. 耗内存，占用内存过高
